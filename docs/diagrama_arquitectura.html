Arquitectura de tres capas del modulo de requerimientos

1. Capa de presentacion

React 18 + Vite + React Router
  • Login y 5 pantallas (Figs. 5a-5e)
  • Tablero Kanban y listado con filtros
  • Control de vistas por rol
  • Cliente HTTP con token JWT
⇄ HTTPS / JSON
REST

2. Capa de logica

Node.js + Express
  • API REST (rutas, controladores)
  • Servicios y reglas de negocio
  • Maquina de estados (Tabla 13)
  • Autenticacion JWT y roles
  • Motor de sugerencia (RF-14)
⇄ SQL
(pool pg)

3. Capa de datos

PostgreSQL 16
  • 11 entidades del modelo logico
  • Catalogos y transiciones
  • Integridad referencial
  • Historial de cambios
Separacion de responsabilidades (RNF-05): cada capa cumple una funcion definida. La interfaz no accede directamente a la base de datos; toda operacion pasa por la API, que aplica las reglas de negocio y el control de acceso antes de persistir o consultar datos. Esta organizacion permite ajustar o ampliar el modulo sin rehacer su base (RNF-07).
